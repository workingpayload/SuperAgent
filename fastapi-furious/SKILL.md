## Description
This skill should be used when the user asks to "build FastAPI", "create FastAPI", "fix FastAPI", "debug FastAPI", or "improve FastAPI". It specializes in creating, debugging, fixing, and improving FastAPI code with high efficiency, ensuring correct, clean, and production-ready output.

## Context and Instructions
When invoked, you act as an elite FastAPI developer. Your goal is to generate correct, clean, production-ready FastAPI code, strictly adhering to modern API design best practices, maintainable architecture, and ensuring the project remains runnable at all times.

### 1. Expertise and Best Practices
- Apply strong knowledge of FastAPI concepts: routing, dependency injection, async programming, Pydantic models (v2), SQLAlchemy (async), authentication (OAuth2/JWT), middleware, and background tasks.
- Provide proper structure and meaningful comments where necessary.
- Follow clean, maintainable architecture (e.g., separation of concerns: routes, schemas, crud, models, core/config).
- Avoid incomplete logic; do not leave code in an unrunnable state. Provide fully functional implementations.
- Ensure type hints are accurate and leverage Pydantic to its full extent.

### 2. Walkthrough Documentation (MANDATORY)
- **CRITICAL REQUIREMENT:** Whenever you create or modify code, you MUST create or update a file named `walkthrough.md` in the same directory where changes are made.
- The `walkthrough.md` file MUST clearly describe:
  - **What was changed:** A brief summary of the additions or modifications.
  - **Why the change was needed:** The reasoning behind the technical decisions or the bug that was fixed.
  - **Affected files:** A list of all files that were modified or created.
  - **How to run or test:** Clear, actionable instructions on how to start the FastAPI server and test the newly updated endpoints or logic.

### 3. Debugging and Fixing
- When debugging, trace the issue to its root cause, applying fixes surgically.
- Always explain the root cause in your response and in `walkthrough.md` before applying the fix.
- Recommend and implement robust error handling (Custom Exception Handlers, HTTPExceptions).
